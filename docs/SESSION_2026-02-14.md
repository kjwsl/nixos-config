## Technical Details

### Custom Plugin Definition
```nix
tmux-powerkit = pkgs.tmuxPlugins.mkTmuxPlugin {
  pluginName = "tmux-powerkit";
  version = "unstable-2026-02-14";
  src = pkgs.fetchFromGitHub {
    owner = "fabioluciano";
    repo = "tmux-powerkit";
    rev = "ccd1c5269964b37c0aab0b284e834c53730c9e4b";
    sha256 = "sha256-zj0jymsNfTifXsSsxcPo4MIAI5o+WAkAXSdPl988BcM=";
  };
  rtpFilePath = "tmux-powerkit.tmux";
};
```

### Configuration
```nix
{
  plugin = tmux-powerkit;
  extraConfig = ''
    set -g @powerkit-enabled true
    set -g @powerkit-theme 'catppuccin'
    set -g @powerkit-status-position 'top'
    set -g @powerkit-plugins 'cpu battery time hostname'
  '';
}
```

## Key Learning: Framework Over Custom Build

### Initial Approach (Wrong)
- Manually configured individual plugins (catppuccin, cpu, battery, etc.)
- Built custom status bar configuration
- Result: User said "it looks ugly and shit"

### User Correction (Right)
User asked: "why can't we just use https://github.com/fabioluciano/tmux-powerkit?"

**Lesson**: When a comprehensive, maintained framework exists, USE IT instead of reinventing.

### Why tmux-powerkit is Better
- ✅ 42 production-ready plugins vs manually configuring 9
- ✅ 37 themes with 61 variants vs single custom theme
- ✅ Smart caching with Stale-While-Revalidate
- ✅ Maintained framework vs custom configuration
- ✅ One-line theme changes vs manual reconfiguration
